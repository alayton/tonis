Overview

Tonis using a dependency container to manage and inject dependencies. Tonis uses a container based on the League Container but any container that implements Interop\Container\ContainerInterface may be used.

For more information on the concepts see Dependency Injection and Inversion of Control on Wikipedia.

Usage

To use a container you must inject it when you create the Tonis\App object.

$container = new \Tonis\Container;
$app       = new \Tonis\App($container);

If no container is specified, Tonis creates the default one for you.

// this is identical to the example above
$app = new \Tonis\App;

The container is available from Tonis\App through the getContainer() method. 

$container = $app->getContainer();
$fooClass  = $container->get('FooClass');

Required Services

If you choose to use your own container you MUST provide the following services:

Tonis\Router
MUST return a NEW instance of Tonis\Router on each get.
Tonis\Handler\ErrorInterface
An instance of Tonis\Handler\ErrorInterface which handles exceptions.
Tonis\Handler\NotFoundInterface
An instance of Tonis\Handler\NotFoundInterface which handles missing routes.
Tonis\View\Manager
An instance of Tonis\View\Manager which handles rendering templates.
